# Issue #1 BossOCR minimal changes v1.0

## 本次更新

- 转发处理函数的所有退出路径都会点击候选人详情页空白区域恢复焦点，包括成功、失败、连续转发上限、无备用邮箱、等待中止和异常路径。恢复坐标为 `(450, 375)`，随机偏移后仍位于规定范围内。
- OCR 最多 8 屏扫描流程保持不变，每次向下滚动幅度从 5–7 格调整为 100–140 格。
- 启动时可输入运行秒数；留空或 `0` 表示持续运行，正整数表示到期自动停止。非交互模式可使用 `--duration-seconds`。
- 关键词规则支持英文双引号关键词、`and`、`or` 和分号分隔的多条规则；`and` 优先于 `or`，OCR 二次确认会复核同一整条规则。

## 明确未包含

- 不包含 P3 已触发转发候选人日志。
- 不包含 P4 关键词数值匹配或比较。
- 不包含 OCR 底部检测、页面状态识别或整体架构重构。

## 下载与使用

1. 下载 Release 附件 `BossOCR-Windows-x64.zip` 和对应 `.sha256` 文件。
2. 校验 ZIP 的 SHA-256 后完整解压，不要只复制 `BossOCR.exe`。
3. 在 Microsoft Edge 登录 BOSS 直聘并打开“推荐牛人”页面。
4. 首次运行请使用 `--no-forward` 安全模式验证 OCR 和关键词规则。

安全模式示例：

```powershell
BossOCR.exe --keywords '"Python"; "短剧"' --no-forward --auto --duration-seconds 60
```

交互模式直接运行 `BossOCR.exe`，按提示输入关键词规则、备用邮箱和运行秒数。运行期间按空格暂停/继续，按 Esc 停止。

旧关键词配置必须手动迁移：

```text
旧：Python;短剧
新："Python";"短剧"
```

## 测试与构建验证

- 发布准备阶段全量单元测试：64 项通过，0 项失败。
- Windows Release 构建会再次运行 `python -m unittest discover -s tests -v`，测试失败时不会打包或发布。
- 构建完成后使用 `--no-forward --auto` 执行低风险启动冒烟测试，不触发真实转发。

## 已知限制

- 仅支持 Windows 10/11 x64，推荐 Python 3.11 构建。
- OCR 只读取当前屏幕可见区域，最多扫描 8 屏，不检测简历底部，允许存在漏看。
- 固定坐标操作受屏幕分辨率、系统缩放、浏览器缩放和页面布局影响。
- 关键词不支持括号、`not`、数值比较、模糊匹配或语义匹配。
- 程序未进行商业代码签名，Windows SmartScreen 可能显示未知发布者。
