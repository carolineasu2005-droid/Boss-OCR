## 实施范围核对

本次需求新增并验证以下能力：

- 新增首位候选人卡片点击区域校准。
- 新增「打开筛选」按钮区域校准。
- 新增「最近没看过」按钮区域校准。
- 新增「筛选确定」按钮区域校准。
- 四个区域全部成功后，才启用 batch filter 自动归位能力。
- 任一区域校准取消或失败时，本次运行禁用 batch filter 自动归位，并回退旧流程。
- 新增 `--no-batch-filter` 禁用开关。
- 启动阶段完成准备动作后再启动运行计时器。
- 启动阶段可自动执行筛选并点击首位候选人。
- 每批 100 人结束后可再次执行筛选并点击首位候选人。

- 未修改 OCR 滚动策略。
- 未修改关键词 `and` / `or` / `not` / `any(...)` parser 和 matcher。
- 未修改邮件转发流程内部逻辑。
- 未修改备用邮箱逻辑。

## 关键流程验收

### 启动阶段流程

已验证启动后可依次校准：

1. 首位候选人卡片区域；
2. 打开筛选按钮区域；
3. 最近没看过按钮区域；
4. 筛选确定按钮区域。

校准完成后，程序可自动执行：

```text
点击「打开筛选」
点击「最近没看过」
点击「筛选确定」
等待候选人列表刷新
点击已校准的首位候选人卡片
进入既有 OCR / 焦点恢复 / 转发点击校准流程
```

验收结果：通过。

### 批次刷新后流程

已验证每批 100 人后可执行：

```text
刷新 / 进入下一批边界
点击「打开筛选」
点击「最近没看过」
点击「筛选确定」
等待候选人列表刷新
点击首位候选人卡片
继续后续 OCR / 关键词 / 转发 / 浏览流程
```

验收结果：通过。

### 鼠标归位能力

本需求消除了旧流程对“人工把鼠标放在首位候选人卡片上”的强依赖。即使鼠标在运行前或批次切换期间被移动，程序仍可通过已校准的首位候选人区域重新点击进入首位候选人。

验收结果：通过。

### `--no-forward` 安全验证

已验证 `--no-forward` 下：

- 可以完成四区域校准。
- 可以自动执行筛选归位。
- 可以进入既有 OCR / 转发相关校准流程。
- 关键词命中时不会真实发送邮件转发。

验收结果：通过。

### `--no-batch-filter` 回退验证

已验证禁用 batch filter 后，程序不启用新增四区域筛选归位流程，旧流程仍可运行。

验收结果：通过。

## 风险核对

已确认：

- 本功能只增加候选人列表筛选和首位候选人归位动作。
- 不检测页面状态。
- 不识别按钮文字。
- 不判断筛选是否真实生效。
- 不读取 DOM。
- 不改变右方向键切换下一位候选人的既有逻辑。
- 不改变转发成功/失败判断。

残余风险：

- 仍依赖当前 BOSS 页面结构和按钮位置。
- 浏览器窗口移动、缩放比例变化或分辨率变化后，需要重新启动并重新校准。
- 如果 BOSS 页面筛选面板结构变化，需重新校准或使用 `--no-batch-filter` 回退。
