## 2. 实施摘要

- 将现有屏幕区域选择器的提示文字参数化，保留 OCR 默认行为。
- 复用 `ScreenRegion` 保存焦点恢复区域。
- 默认区域实现为 `left=400, top=350, width=101, height=51`，按半开区间语义严格覆盖 `X:400-500, Y:350-400`。
- 新增仅在进程当前运行期间生效的区域、请求、尝试和进行中状态。
- 普通交互模式启动时询问是否校准；实际框选发生在第一位候选人详情页打开后、首次浏览前。
- `--auto` 和现有命令行非交互路径不询问、不弹出新增的焦点恢复区域框选层，并使用默认区域。
- 校准取消或异常时显式恢复默认区域，不停止浏览、OCR 或转发流程。
- `forward_one_candidate()` 的统一 `finally` 从运行期区域取点，并调用 `human_click(..., offset=0)`。
- README 已补充交互、回退、安全验证和限制说明。

## 7. 风险与限制

- 校准结果是绝对屏幕坐标；校准后移动、缩放或最大化状态变化会使区域失效。
- 第一版区域框选仍只支持主显示器；混合 DPI 多显示器未覆盖。
- 用户可能把按钮、链接或其他交互控件框入空白区域，程序不做 DOM 或视觉状态判断。
- 系统缩放、浏览器缩放、页面布局更新和遮挡仍可能影响坐标准确性。
- `--auto` 使用默认区域，无法适应不同窗口位置；运行前需确认默认区域安全。
- 其余邮件转发步骤仍依赖固定屏幕坐标。
- `--no-forward` 不调用真实转发函数，因此只能安全验证询问、框选、OCR 和浏览流程，不能验证真实转发后的页面焦点效果。
- 自动化测试无法完全替代 Windows Edge 真实界面验证。

### 8.2 建议的安全验证步骤

1. 在主显示器打开 Windows Edge，并登录测试账号或受控环境。
2. 先以普通交互方式启动 `--no-forward`，不要使用 `--auto`：

   ```powershell
   .\venv\Scripts\python.exe simple_brush.py --no-forward
   ```

3. 输入只用于测试的关键词规则。
4. 在焦点恢复校准询问中选择 `y`。
5. 打开第一位候选人详情后，框选明确无按钮、链接和输入控件的空白区域。
6. 验证 Esc 取消时程序继续运行并采用默认区域。
7. 验证正常框选时区域只在本轮有效，重启后恢复默认状态并重新询问。
8. 使用 `--auto` 做一次安全运行，确认不会出现新增的焦点恢复询问或框选层，并确认当前窗口布局下默认区域安全。
9. 查看日志，确认无校准异常或 OCR 错误。

如果必须验证真实转发后的焦点恢复，只能在测试账号、测试邮箱、小规模样本和人工全程监控下进行；首次只验证一位候选人，确认目标区域安全后立即停止。不得直接进行无人值守或批量真实转发。
